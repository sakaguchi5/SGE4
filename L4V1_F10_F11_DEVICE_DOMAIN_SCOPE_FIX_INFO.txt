SGE4 L4V1-F10-F11 DeviceDomain scope repair

Cause:
The previous patch detector searched the entire D3D12Backend.cpp file. Legacy per-Package Instance recovery owns the same removed-adapter fragments, so the detector could attach the F10 marker to DeviceDomain without actually patching DeviceDomain.

Fix:
- Apply-Level4v1F10F11BackendPatch.ps1 now extracts only the DeviceDomain class and repairs each required fragment inside that scope.
- Verify-Level4v1F10F11Boundaries.ps1 now verifies only the DeviceDomain class scope.
- 46J prints all Retry qualification fields if exit code 14 occurs again.

Apply:
1. Overwrite these files at the SGE4 repository root.
2. Run .\prepare_sge4_level4v1_f10_f11.bat
3. Run .\run_sge4_level4v1_f10_f11.bat

Existing generated objects and prior Backend edits do not need to be reverted.
