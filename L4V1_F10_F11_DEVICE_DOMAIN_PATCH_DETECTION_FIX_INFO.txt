SGE4 L4V1-F10-F11 DeviceDomain patch detection fix

Cause:
The original Apply script used a global hasExcludedAdapterLuid_ substring as its idempotency test.
That name already existed in the legacy per-Package Instance, so the script falsely skipped the new DeviceDomain patch.

Fix:
- Use a DeviceDomain-specific qualification marker and four semantic fragments.
- Detect and reject partial application.
- Keep repeated prepare runs idempotent.
- Make the boundary gate verify the DeviceDomain marker and whitespace-tolerant semantic patterns.
