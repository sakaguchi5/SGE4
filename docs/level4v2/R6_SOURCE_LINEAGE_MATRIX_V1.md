# R6 source lineage matrix V1

R6 imports exactly the sixteen source identities frozen by R0. It imports their identities and proof roles, not mutable file contents or executable authority.

| Role | R6 use |
|---|---|
| Level4V1ReconstructionPolicy | Composition and architecture scope |
| Level4V1RuntimeRecoveryQualification | Runtime, WARP and whole-composition Recovery lineage |
| Spiral1ProofLedger | Semantic/representation separation and Candidate lineage |
| Spiral2ProofLedger | Composition/representation crossing lineage |
| Spiral3ProofLedger | Planner-independent authority lineage |
| Spiral4ProofLedger | Explicit invocation and verified indirect quantity |
| Spiral5ProofLedger | Temporal History validity |
| Spiral6ProofLedger | Exact sparse membership |
| Spiral7ProofLedger | Sparse-temporal delta, epoch and RecoverySeed rules |
| Spiral7Closure | No Runtime policy and external Evidence retention |
| AcceptedArchitectureQualification | Correctness and Runtime/Recovery qualification |
| AcceptedObservationalMeasurement | Observational lane only |
| FinalDecisionSummary | Paired Decision authority and no universal winner |
| CanonicalInputNarrative | Reconstruction boundaries and prohibitions |
| CapabilityMatrix | Proven capability scope |
| RetainedReferenceLayout | No deletion before migration proof and R7 decision |

The path and SHA-256 for each role are recorded in `R6_MIGRATION_CORPUS_MANIFEST_V1.json` and must match the R0 source freeze.
