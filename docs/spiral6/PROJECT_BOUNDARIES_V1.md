# Spiral 6 project boundaries V1

## Semantic authority

Owns the fixed 4096-element universe, canonical point-transform meaning, exact set identity and inactive sentinel.

## Candidate Planner

May propose dense-mask, compact-index or active-block lowering fields. It cannot mint Verified authority.

## Independent Verifier

Must not depend on the Planner. It independently reconstructs all three representations and the exact legal output write set.

## Compiler / Frozen artifact

May lower only an opaque Verified Sparse Plan. Frozen bytes bind the set, Candidate, representation role, Target and Program identities.

## Runtime / Backend

Mechanically binds verified Resources and submits verified Dispatch operations. It may not choose membership, cardinality, pattern, Candidate, compaction or output write indices.

## Recovery

Invalidates representation Resources and set bindings. A new epoch requires explicit set rebind, representation rebuild and sentinel initialization.

## Hardware and game boundary

Hardware is Target/Measurement evidence. Game-specific selection of the exact set occurs outside SGE and is not optimized by Spiral 6.
