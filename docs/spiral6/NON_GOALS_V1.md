# Spiral 6 V1 non-goals

Spiral 6 V1 does not infer or implement game policy. In particular, it does not decide visibility, culling, distance, priority, LOD, or importance.

It also excludes:

- approximate or probabilistic omission,
- dynamic Resource capacity,
- compacted output order,
- temporal history comparison,
- variable hierarchy or representation path,
- Texture/Raster/meshlet/BVH specialization,
- streaming and residency,
- multiple DeviceDomains or partial Recovery,
- Runtime adaptive Candidate selection,
- a universal claim that sparse execution is faster.

The exact set is external verified input. Sparse Candidates may only change its physical representation and execution granularity.
